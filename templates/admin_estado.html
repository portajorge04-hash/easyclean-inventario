Guía de respaldo
  1. Haz clic en Descargar Respaldo (JSON) para guardar todos los datos.
  2. Guarda el archivo en Google Drive o en tu PC.
  3. Recomendado: hacer respaldo una vez por semana.
  4. Si algo falla, el archivo JSON contiene todos los lotes, compras, movimientos y stocks.
Cómo están protegidos los datos
Base de datos separada
Los datos están en Neon PostgreSQL, completamente separado del servidor Railway. Un nuevo deploy de código nunca toca la base de datos.
Migraciones controladas
Cada cambio de estructura se registra. Si ya se aplicó, nunca se ejecuta de nuevo. Imposible duplicar o borrar datos por un deploy.
Solo se agregan tablas
El código nunca ejecuta DROP TABLE ni TRUNCATE. Las tablas nuevas se crean con IF NOT EXISTS, sin afectar las existentes.
Respaldo descargable
En cualquier momento puedes descargar un JSON con todo el historial: lotes, compras, movimientos y stocks actuales.
