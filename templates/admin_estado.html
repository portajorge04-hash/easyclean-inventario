⚠️ ATENCIÓN: El sistema está usando SQLite (base de datos local)

Los datos se borrarán cada vez que Railway haga un redeploy. Esto ocurre porque la variable de entorno DATABASE_URL no está configurada en Railway.

Para solucionar esto de forma permanente:

  1. Ve a railway.app → tu proyecto → pestaña Variables
  2. Agrega la variable DATABASE_URL con la URL de tu base de datos PostgreSQL (Neon, Supabase, etc.)
  3. Railway redesplegará automáticamente y los datos quedarán en la nube de forma permanente

Mientras tanto, el sistema tiene una guardia automática que restaura los datos del backup del 2026-05-04 cada vez que detecta que fueron borrados. Usa el botón de abajo para restaurar manualmente.
